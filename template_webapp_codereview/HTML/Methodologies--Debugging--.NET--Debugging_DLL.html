Debugging DLL


Steps to debug a specific DLL for .NET in IIS

1. Save a local copy of the DLL
2. Modify DLL adding debug assembly attributes and save module using DnSpy
3. Close all modules
4. Attach selected process (for IIS, w3wp.exe)
5. Debug -> Break All
6. Debug -> Windows -> Modules
7. Right-click DLL and click "Open all modules"
8. Sort Assemblies
9. Navigate to specific DLL and namespace
10. Set breakpoint at desired function
11. Use application and expect to hit breakpoint
12. Look for call stack references
13. Check for locals and other variables
