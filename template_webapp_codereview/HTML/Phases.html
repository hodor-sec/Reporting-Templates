Phases



Setup
- Setup environment and prerequisites to analyze targeted application
- Check native libraries related to application
- Extract sourcecode by extracting / copying / decompiling
- Enable error and DBMS logging on debugging machine

Initial vulnerability discovery
- Analyze source code for initial vulnerability discovery
- Search for specific keywords by regex
- Check unauthenticated areas, look for keywords indicating public access is allowed
- Check authenticated portions of applications, having less focus
- Check sanitization of user input; custom and/or trusted open-source libraries/code
- Follow application logical flow
- Check language specific HTTP request handlers
- Keep track of related classes/functions/methods and their references
- Use regex to search for referenced methods and classes

Vulnerability and dynamic analysis
- Vulnerability analysis of targeted application
- Attempt to trigger the vulnerability and check results in logging or output
- Check for filtered/translated characters during input
- Check DBMS logging for specific syntax errors
- Attempt to manually enter entire query in DBMS CLI for analysis
- Dynamic / Debug analysis of targeted compiled application
- Java servlets: check processes via Process Explorer and viewing properties
- Java: parse web.xml for servlets
- Java: look for native JAR servlets in WEB-INF folder
-.NET: use DnSpy and set breakpoints on interesting lines

Triggering vulnerability
- Create a POC to successfully trigger, reproduce and to exploit the vulnerability
- Use debug statements in interpreted / scripted code

Bypassing filters
- Modify POC to bypass common filters, if required

Exfiltration
- Perform actions which attempt to achieve code execution, such as:
- Exfiltrate data for enumeration of environment
- Creating an admin user
- Facilitating a reverse shell
- Etc.

Command execution
- Create a fully automated one-shot RCE script
